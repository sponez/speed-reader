# Speed Reader: информация о проекте

## Идея

Speed Reader - личное Windows desktop-приложение для тренировки скорости чтения методом скользящего указателя.

Пользователь вставляет текст, выбирает параметры видимости и целевую скорость чтения, после чего запускает попытку. Во время чтения приложение подсвечивает текущий фрагмент текста, двигает окно видимости с заданной скоростью, а остальной текст размывает. Цель - помочь глазам двигаться по тексту равномернее и снизить количество возвратов назад.

## Целевая аудитория

Первая версия делается для личного использования:

- один пользователь;
- Windows как основная платформа;
- локальный запуск без сервера, аккаунтов и синхронизации;
- простой сценарий без лишней статистики и отвлекающих элементов.

## MVP-сценарий

1. Пользователь открывает приложение.
2. Вставляет текст для чтения.
3. Настраивает параметры:
   - целевую скорость в WPM;
   - размер окна видимости как количество слов вокруг текущего слова;
   - силу размытия остального текста.
4. Запускает чтение.
5. Приложение скрывает подготовительный интерфейс и показывает только режим чтения.
6. Текущее слово подсвечивается, окно видимости движется по словам согласно WPM.
7. Всё вне окна видимости размывается.
8. Пользователь нажимает `Esc`, чтобы завершить попытку и вернуться к подготовке.

Во время чтения нет паузы, изменения настроек, статистики, истории сессий и дополнительных панелей.

## Стек

Рекомендуемый стек для первой версии:

- **Tauri 2** - desktop-оболочка для Windows с Rust-бэкендом и WebView2-рендерингом.
- **Vite 7** - быстрый dev-сервер и сборка фронтенда.
- **React 19** - компонентная модель для интерфейса подготовки и режима чтения.
- **TypeScript 5** - типизация доменной логики, настроек и состояния чтения.
- **npm** - package manager, потому что он уже установлен локально.
- **Tauri Store** - локальное сохранение настроек и последнего текста в данных приложения.

### Почему этот стек

Основная сложность проекта находится в отрисовке: подсветка, размытие, измерение текста, плавное движение окна видимости. Веб-рендеринг хорошо подходит для этого, потому что CSS и DOM дают готовые инструменты для текста, `filter: blur(...)`, позиционирования и анимаций.

Tauri даёт desktop-приложение без тяжёлой Electron-оболочки и позволяет оставить Rust-часть минимальной: запуск окна, сборка приложения, доступ к локальному хранилищу.

### Что не берём в MVP

- **Electron** - проще по экосистеме, но тяжелее для личного небольшого приложения.
- **Pure native UI** - может дать нативность, но усложнит работу с текстовым layout, blur и анимациями.
- **Backend/server** - не нужен, потому что приложение локальное и персональное.
- **Аккаунты и синхронизация** - не входят в первый сценарий.
- **История чтения и аналитика** - отложены, чтобы режим чтения оставался максимально спокойным.

## Основные настройки MVP

- `wpm` - целевая скорость чтения в словах в минуту.
- `visibleWordsBefore` - сколько слов до текущего остаётся в окне видимости.
- `visibleWordsAfter` - сколько слов после текущего остаётся в окне видимости.
- `blurIntensity` - сила размытия текста вне окна видимости.
- `lastText` - последний вставленный текст.

## Текущее состояние scaffold

Проект уже инициализирован как Tauri 2 + Vite + React + TypeScript приложение.

Фактические параметры:

- package name: `speed-reader`;
- product name: `Speed Reader`;
- Tauri identifier: `com.local.speedreader`;
- dev URL: `http://localhost:1420`;
- frontend output: `dist`;
- Tauri source: `src-tauri`;
- package manager: `npm`.

Основные npm scripts:

- `npm run dev` - запуск Vite dev-сервера;
- `npm run build` - TypeScript check и Vite production build;
- `npm run preview` - preview production frontend;
- `npm run tauri` - Tauri CLI wrapper.

4. Реализовать доменную токенизацию текста на слова и разделители.
5. Реализовать расчёт текущего слова по WPM и прошедшему времени.
6. Реализовать режим чтения без отвлекающих элементов.
7. Добавить подсветку текущего слова и размытие текста вне окна видимости.
8. Добавить завершение попытки по `Esc`.
9. Подключить Tauri Store для сохранения настроек и последнего текста.
10. Написать базовые unit-тесты для токенизации и расчёта прогресса.
11. Добавить UI/smoke-проверки ключевого сценария.
12. Собрать Windows installer через Tauri build.

## Тестовый минимум

- Токенизация корректно обрабатывает пробелы, переносы строк, пунктуацию и пустой текст.
- Расчёт WPM корректно переводит прошедшее время в индекс текущего слова.
- Настройки и последний текст сохраняются и загружаются.
- После запуска подготовительный UI скрывается.
- `Esc` завершает попытку чтения.
- Активное слово подсвечено, окно видимости не размыто, остальной текст размыт.
